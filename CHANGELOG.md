# Changelog

All notable changes to the EndureIT project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- Initial project setup with monorepo structure
- FastAPI backend with comprehensive fitness tracking features
- Workout tracking with multiple workout types
- Goal setting and progress monitoring
- Body measurements and progress tracking
- Dashboard with aggregated statistics
- JWT authentication system
- Service layer architecture for business logic
- PostgreSQL database with SQLAlchemy ORM
- Redis caching layer
- Nginx reverse proxy
- Docker and Docker Compose orchestration
- Comprehensive documentation (README, SETUP, QUICK_START, etc.)
- GitHub Actions CI/CD workflows
- EditorConfig for consistent code formatting

### Backend Features
- User authentication and authorization
- 9 workout types (Running, Cycling, Swimming, Gym, etc.)
- Exercise tracking with sets, reps, and weights
- Performance metrics (heart rate, pace, calories, elevation)
- Goal management with progress tracking
- Body composition tracking
- Dashboard analytics
- 30+ API endpoints
- Interactive API documentation (Swagger/ReDoc)

### Infrastructure
- Docker containerization
- PostgreSQL 15 database
- Redis 7 cache
- Nginx reverse proxy
- Health checks for all services
- Development and production Docker configurations

### Documentation
- Root README with project overview
- Backend-specific documentation
- Setup guide with detailed instructions
- Quick start guide for daily use
- Project structure documentation
- Contributing guidelines
- Architecture overview
